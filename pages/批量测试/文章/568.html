cr880X刷AX3000方法分享（以下用CR8806示范）：
资源文件回复可见，分区文件是：AX3000原厂分区
好处是：可以与米家的路由器自由自在组mesh、有插件管理、解除营运商接入限制。
坏处是：2.4G功率减半
操作有风险！先把目前的分区备份一下作后悔用途谢谢！
TTL法：
cr880X成功进入uboot中断后，把红米AX3000的三个分区文件放到Tftpd64目录中，双击Tftpd64运行（运行后不要管它）。
输入以下命令，并手动将网卡ip修改成192.168.1.1，网关为192.168.1.2
setenv ipaddr 192.168.1.2
setenv serverip 192.168.1.1

以上设置好之后，输入以下命令：
tftpboot OpenWrt.mtd1.bin && flash 0:MIBIB
做完不要断电！继续输入以下命令：
tftpboot OpenWrt.mtd11.bin && flash 0:APPSBL
tftpboot OpenWrt.mtd12.bin && flash 0:APPSBL_1
确认好步骤，没有报错后直接拔电！
然后通电后会发现闪黄灯，别慌！拔电，长按reset不松手插电用MIWIFIRepairTool刷红米AX3000_1.0.33版救机包！
待蓝灯闪烁后拔电，关闭MIWIFIRepairTool软件即可！

================================================================================
如现时固件能进ssh，可以直接在ssh上操作

ssh法：
如果已刷openwrt想强刷红米AX3000官方固件，也行！
在ssh环境，把这三个分区刷回去就行，记住途中不要断电，要连续把三个文件刷回去，检查无漏无误才断电！
先用winscp把三个文件上传到/tmp/，然后用putty操作：

命令如下：
mtd erase /dev/mtd1
mtd write /tmp/OpenWrt.mtd1.bin /dev/mtd1
做完不要断电！继续输入以下命令：
mtd erase /dev/mtd11
mtd write /tmp/OpenWrt.mtd11.bin /dev/mtd11
mtd erase /dev/mtd12
mtd write /tmp/OpenWrt.mtd12.bin /dev/mtd12
完成检查无漏无误后拔电！刷机风险需自行承当！
然后通电后会发现闪黄灯，别慌！拔电，长按reset不松手插电用MIWIFIRepairTool刷红米AX3000_1.0.33版救机包！
待蓝灯闪烁后拔电，关闭MIWIFIRepairTool软件即可！

以下是TTL环境刷Openwrt：

回形针接线TTL免拆壳大法swfighter大佬：记CR8808不拆机刷集客AP过程，在8806固件下开启ssh-小米无线路由器以及小米无线相关的设备-恩山无线论坛 (right.com.cn)
vcc不用接！也不要接！记住：接了VCC SOC会停机，严重会炸soc ！炸电源！

以下是cr880x ax3000脚位参考图






为了方便，这里我焊接了2.5mm排针


和模块接好线后，打开PuTTY (64-bit)软件（路由器不要提前插电!）


设置ttl模块端口和速率
请各位按照实际设置，然后按open


收到大量反映，乱码的、黑屏的，可能是TTL设置不正确






弹出ttl界面，然后路由器插电不断按回车，让路由器转跳到U-boot
（如果显示乱码，把TX、RX线调换即可，波比率速率调至115200！）
（如果还是黑屏请检查com端口是否正确，还有路由器不要提前插电！还有VCC不要接！）
（这一步做不了的需要下载救机包，刷机后即可继续）



输入以下令命：
setenv boot_wait on
setenv uart_en 1
saveenv



然后把下载的固件放在tftpd64软件目录，双击tftpd64运行（打开后无需设置理会）


使用网线连接网卡和路由器lan口，手动设置网卡ipv4地址


输入以下命令：
setenv ipaddr 192.168.1.2
setenv serverip 192.168.1.1
tftpboot （你下载的ubi固件文件名！）


下载完成后继续输入以下命令进行刷写：
flash rootfs_1
setenv flag_try_sys2_failed 0
setenv flag_boot_rootfs 1
setenv flag_last_success 1
saveenv

改过大分区救机的，要用以下代码：
如果上面代码没有反馈错误就不要输入下面的代码！！
flash rootfs
setenv flag_try_sys1_failed 0
setenv flag_boot_rootfs 0
setenv flag_last_success 0
saveenv



继续输入以下命令进行重启：
reset


重启后ttl信息显示op在跑码 在22附近左右会卡很久（等就行）


在最后会显示6秒后自动重启！
这时一定要拔掉ttl插线和网线！不然会无限重启
待蓝灯不再闪烁时刷机完成！
然后ipv4设置自动获取！再输入192.168.1.1进入管理界面！


无线电默认是关闭的，需要插网线！
先打开无线电，生效后再设置无线电名字和密码！
wifi 5g正常  2.4g可以用但信号低


固件默认是没有中文的，到software进行安装(这步要连网！)



安装完成后刷新页面即可

救机包刷机大法
想用回小米官方固件直接使用MIWIFIRepairTool刷救机包即可！

或者无法进入uboot，请下载对应救机包，并使用 MIWIFIRepairTool 进行刷机
(按住reset键不松手插电，直到橙灯闪烁进入救机模式！)

如果使用MIWIFIRepairTool刷机进入无限断线问题，是网卡问题
（本人使用219-v网卡就遇到这个问题，后来用螃蟹网卡救机秒认！）
救机路由器默认ip地址是192.168.31.1 客户端是192.168.31.100
请注意，不对是不可能救机！或者用自动获取ipv4方式让路由器自行DHCP进行救机！

大分区不死界面刷入教程,本教程使用CR8806进行
（勇者专属区，翻车自负）
自认不是大佬大神、对分区不了解，对TTL、UBOOT反馈信息不明不白的请不要尝试

刷完后TTL不再有输出！想恢复TTL请刷写前备份好mtd11分区！
注：集客的mtd11 mtd12分区好像是反的？

第一步，按照上面的TTL教程刷入op
第二步，使用putty，ssh方式连接op

第三步，输入命令cat /proc/mtd确认分区信息

确认好mtd1 mtd11 mtd12和对应刷入文件是否正确
（为什么要确认？因为集客好像是反的！）

第四步，使用winscp上传MIBIB.bin文件到/tmp/

第五步，输入mtd erase /dev/mtd1（清除mtd1分区数据）

第六步，输入mtd write /tmp/MIBIB.bin /dev/mtd1（MIBIB.bin写入到mtd1）

第七步，确认写入无报错，直接拔电！等两秒后插入电源，等op启动完成

第八步，继续用puttty登录ssh

第九步，继续用winscp把APPSBL.bin APPSBL1.bin文件上传到/tmp/

第十步，输入mtd erase /dev/mtd11（清除mtd11分区数据）

第11步，输入mtd write /tmp/APPSBL.bin /dev/mtd11（APPSBL.bin写入到mtd11）

第12步，输入mtd erase /dev/mtd12（清除mtd12分区数据）

第13步，输入mtd write /tmp/APPSBL1.bin /dev/mtd12（APPSBL1.bin写入到mtd12）

第14步，确认写入无报错，直接拔电！

第15步，用牙签顶住reset不松手插电保持10秒，192.168.10.1进入不死界面！

第16步，直接刷本教程的2月11号OP固件！


成功进入OP系统，你会发现变成大分区！

完成！
